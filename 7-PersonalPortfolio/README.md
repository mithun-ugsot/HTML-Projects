# Personal Portfolio Project - HTML Fundamentals

## 1. Problem Statement

Create a comprehensive personal portfolio website using **only HTML** (no CSS or JavaScript) that demonstrates fundamental HTML concepts and best practices. The portfolio should showcase a software engineer's profile, projects, skills, and contact information in a semantically correct and accessible manner suitable for a 20-minute presentation.

---

## 2. Tech Stack

| Category | HTML Concepts |
|----------|---------------|
| **Document Structure** | DOCTYPE, `<html>`, `<head>`, `<body>`, Meta tags, `<title>` |
| **Semantic Elements** | `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>` |
| **Text Formatting** | `<h1>`-`<h6>`, `<p>`, `<strong>`, `<em>`, `<small>`, `<mark>`, `<cite>`, `<abbr>`, `<time>` |
| **Lists** | `<ul>`, `<ol>`, `<dl>`, `<dt>`, `<dd>`, Nested lists |
| **Links** | `<a>` with href, anchor links, `target`, `rel` attributes, `mailto:` |
| **Code & Quotes** | `<pre>`, `<code>`, `<blockquote>` |
| **Tables** | `<table>`, `<caption>`, `<thead>`, `<tbody>`, `<tfoot>`, `<tr>`, `<th>`, `<td>`, `colspan` |
| **Forms** | `<form>`, `<label>`, `<input>`, `<select>`, `<textarea>`, `<button>`, `<fieldset>`, `<legend>` |
| **Interactive** | `<details>`, `<summary>` |
| **Other** | `<address>`, `<hr>` |

**Total Concepts Covered:** 30+

---

## 3. Expected Outcomes

1. Master HTML5 document structure, semantic elements, and proper heading hierarchy
2. Implement accessible forms with various input types, labels, and validation attributes
3. Create well-structured tables and all three list types (ordered, unordered, definition)
4. Apply SEO best practices using meta tags and semantic markup
5. Understand and implement web accessibility standards with proper element associations
6. Develop clean, maintainable HTML code following industry best practices

---

## 4. Additional Notes

- **Content Reference:** Portfolio based on Linus Torvalds (Linux creator) for technical relevance
- **No Styling:** Pure HTML only - focuses on structure and semantics, not presentation
- **Educational Focus:** Designed for 20-minute course demonstration with maximum HTML concept coverage
